# CS2 Demo Analytics

Локальный инструмент аналитики CS2 демо (`.dem`) на FastAPI + pandas + SQLite + vanilla JS.

## Что исправлено в этой версии

- Реальные и синтетические данные помечаются источником (`demoparser2` / `synthetic_fallback`).
- В аналитике можно выбрать любого игрока матча.
- Показывается информация о матче: файл, карта, количество раундов, длительность, источник.
- Количество раундов считается из данных раундов, а не захардкожено.

## Быстрый запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Открыть: `http://localhost:8000`

## Использование

1. Выберите `.dem` файл.
2. Нажмите **Анализировать**.
3. После загрузки:
   - проверьте блок **Информация о матче**,
   - выберите игрока в выпадающем списке,
   - смотрите метрики, таблицу раундов и графики.

## API

- `POST /api/upload` — загрузка демо.
- `GET /api/matches` — список матчей.
- `GET /api/matches/{match_id}` — аналитика матча.
- `GET /api/matches/{match_id}?player_steam_id=<steam_id>` — аналитика конкретного игрока.

## Проверка корректности данных

- Сумма `kills` по таблице раундов должна совпадать с `overview.kills` для выбранного игрока.
- Количество строк в таблице раундов должно совпадать с `match_info.total_rounds`.
- При смене игрока в селекторе значения должны изменяться под этого игрока.

## Важно

Если `demoparser2` не может корректно распарсить файл, включается `synthetic_fallback`.
Это позволяет проверить работу интерфейса и пайплайна локально, но такие цифры не являются реальной статистикой матча.
